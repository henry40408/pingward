# E2E Testing Harness (Playwright + BDD) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a browser-level E2E suite (Playwright + playwright-bdd) that drives a real compiled `pingward` binary to verify the authentication flows (setup / login / logout).

**Architecture:** A self-contained Node project under `e2e/`, modeled on the sibling `rdrs` project. `global-setup.js` builds the debug binary once. Each scenario spawns its own instance of that binary against a fresh temp SQLite DB on a random port (per-scenario isolation, forced by pingward's one-time `/setup` and lack of open registration), polls `GET /healthz` for readiness, then drives Chromium through the real UI. Gherkin `.feature` files compile to Playwright specs via `playwright-bdd`.

**Tech Stack:** Node 22 (ESM), `@playwright/test` 1.61.1, `playwright-bdd` 9.2.0, Chromium. App under test: Rust / axum / sqlx (SQLite).

## Global Constraints

- **Dependency cooldown:** No package version published less than 7 days ago. Pinned versions verified 2026-07-15: `@playwright/test@^1.61.1` (published 2026-06-23), `playwright-bdd@^9.2.0` (published 2026-06-18). If re-resolved later, pick the latest release ≥7 days old.
- **Selectors:** originally used existing DOM (`#username`, `#password`, button text) with no app-code changes. **Amended 2026-07-15** (post-implementation, at the user's request): templates gained additive `data-testid` hooks (`username-input`, `password-input`, `setup-submit`/`login-submit`, `setup-error`/`login-error`, `logout-button`) and steps use `getByTestId`, since future elements may lack an `id`. Existing `id`s are kept for `<label for>`. Rebuild the binary (`cargo build`) after template edits — askama embeds templates at compile time. See the spec's §11.
- **Binary spawned with `cwd` = repo root.** sqlx migrations use relative paths (`migrations/sqlite`), so the working directory must be the repo root.
- **CI actions pinned to commit SHA** with a `# vX` comment, matching `.github/workflows/ci.yml`.
- **Git:** commits GPG-signed (default config). Stage files explicitly by name — never `git add -A`/`.`.
- **All GitHub-facing text in English.**

## Deviations from spec

- The spec's negative scenario "Setup rejects empty credentials" is **dropped**. The setup/login templates mark inputs `required`, so a browser blocks empty submission client-side; the server's empty-field branch is unreachable through a real browser without contriving a bypass. The `store`-level and `axum-test` layers already cover that branch. Net first-cut scenarios: 5.

- [ ] **Step 6: Create the server harness**

Create `e2e/support/server.js`:

```javascript
import { spawn } from "node:child_process";
import { mkdtempSync, rmSync } from "node:fs";
import { tmpdir } from "node:os";
import net from "node:net";
import path from "node:path";
import { fileURLToPath } from "node:url";

// support/server.js -> e2e/support -> e2e -> repo root
const repoRoot = path.resolve(
  path.dirname(fileURLToPath(import.meta.url)),
  "..",
  ".."
);
const BINARY = path.join(repoRoot, "target", "debug", "pingward");

function findAvailablePort() {
  return new Promise((resolve, reject) => {
    const srv = net.createServer();
    srv.unref();
    srv.on("error", reject);
    srv.listen(0, "127.0.0.1", () => {
      const { port } = srv.address();
      srv.close(() => resolve(port));
    });
  });
}

async function waitForServer(url, timeoutMs = 30000) {
  const deadline = Date.now() + timeoutMs;
  while (Date.now() < deadline) {
    try {
      const res = await fetch(`${url}/healthz`);
      if (res.ok) return;
    } catch {
      // server not up yet
    }
    await new Promise((r) => setTimeout(r, 100));
  }
  throw new Error(`pingward did not become ready at ${url} within ${timeoutMs}ms`);
}

export async function spawnPingward() {
  const dir = mkdtempSync(path.join(tmpdir(), "pingward-e2e-"));
  const dbPath = path.join(dir, "test.sqlite3");
  const port = await findAvailablePort();
  const url = `http://127.0.0.1:${port}`;

  const proc = spawn(BINARY, [], {
    cwd: repoRoot, // sqlx migrations use relative paths
    env: {
      ...process.env,
      DATABASE_URL: `sqlite://${dbPath}?mode=rwc`,
      PINGWARD_BIND: `127.0.0.1:${port}`,
      RUST_LOG: "warn",
    },
    stdio: "ignore",
  });

  const cleanup = async () => {
    proc.kill("SIGTERM");
    rmSync(dir, { recursive: true, force: true });
  };

  try {
    await waitForServer(url);
  } catch (err) {
    await cleanup();
    throw err;
  }

  return { url, dbPath, cleanup };
}
```

- [ ] **Step 1: Create the API helper**

Create `e2e/support/api.js`:

```javascript
// pingward has no open registration: the only bootstrap path is the one-time
// POST /setup (first admin), which is CSRF-exempt (pre-session). We POST the
// form and follow the redirect; a 2xx result means the admin was created.
// Used only against a fresh server, so the empty-field / already-set-up
// re-render branches never fire here.
export class ApiHelper {
  constructor(baseUrl) {
    this.baseUrl = baseUrl;
  }

  async bootstrapAdmin(username, password) {
    const body = new URLSearchParams({ username, password });
    const res = await fetch(`${this.baseUrl}/setup`, {
      method: "POST",
      headers: { "content-type": "application/x-www-form-urlencoded" },
      body,
    });
    if (!res.ok) {
      throw new Error(
        `bootstrapAdmin failed for "${username}": HTTP ${res.status}`
      );
    }
  }
}
```
